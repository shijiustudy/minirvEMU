# minirvEMU: 简易 RISC-V 处理器模拟器

本项目是一个基于 C 语言实现的 minirv 处理器模拟器（`minirvEMU_v2.c`）。项目在软件层面上还原了处理器的取指、译码、执行以及访存逻辑，完整实现了 minirv 规范中要求的 8 条核心指令，用于模拟执行基础的 C 程序二进制文件。

## 目录结构

项目的所有测试数据（由 C 程序编译而来的 `.hex` 文本文件及对应的反汇编 `.txt` 文件）统一存放在 `logisim-bin` 目录中：

```text
.
├── logisim-bin
│   ├── mem.bin
│   ├── mem.hex
│   ├── mem.txt
│   ├── sum.bin
│   ├── sum.hex
│   ├── sum.txt
│   └── vga.txt
├── minirvEMU_v2
└── minirvEMU_v2.c

1 directory, 9 files
```

## 支持的指令集 (共 8 条)

本模拟器完整实现了以下 8 条 RISC-V 基础指令：

### 算术与逻辑指令
1. **`add`** (R-Type)：标准的寄存器加法操作。
2. **`addi`** (I-Type)：立即数加法操作，将源寄存器的值与符号扩展后的立即数相加。
3. **`lui`** (U-Type)：加载高位立即数。处理 U 型立即数格式，将 20 位立即数写入目的寄存器的高 20 位。

### 控制流指令
4. **`jalr`** (I-Type)：间接跳转指令。将下一条指令的地址 (`PC + 4`) 写入目的寄存器，并将 `PC` 设置为源寄存器的值与符号扩展后的立即数相加的结果。

### 访存指令 (Load/读取)
5. **`lw`** (I-Type)：从内存读取一个 32 位的字。本实现假设所有 `lw` 访存都是对齐的（即地址的最低 2 位为 0），不处理不对齐异常。
6. **`lbu`** (I-Type)：从内存读取一个无符号字节并进行零扩展。通过判断访存地址的偏移量，从 32 位读出数据中截取对应的目标字节。

### 访存指令 (Store/写入)
7. **`sw`** (S-Type)：向内存目标地址写入一个完整的 32 位字。
8. **`sb`** (S-Type)：向内存写入一个字节。通过内部生成字节写使能（Byte Enable）掩码，确保只更新 32 位存储字中的特定 8 位，而不破坏其他字节。

## 数据文件与格式转换

Logisim 等硬件工具导出的内存镜像通常为十六进制文本文件（`.hex`）。为了让 C 语言模拟器能够直接读取机器码，需要将其转换为小端序的纯二进制文件（`.bin`）。

请在终端中使用以下命令进行转换（以 `sum.hex` 为例）：

```bash
tail -n +2 logisim-bin/sum.hex | sed -e 's/.*: //' -e 's/ /\n/g' | sed -e 's/\(..\)\(..\)\(..\)\(..\)/\4 \3 \2 \1/' | xxd -r -p > logisim-bin/sum.bin
```

## 编译与运行方式

模拟器采用 C 语言编写，使用标准的 GCC 进行编译。程序的指令与数据采用统一编址，加载同一个 `.bin` 文件即可运行。

1. **编译模拟器**：
   ```bash
   gcc minirvEMU_v2.c -o minirvEMU_v2
   ```

2. **运行测试程序**（通过命令行参数传入 `.bin` 文件路径）：
   ```bash
   ./minirvEMU_v2 logisim-bin/sum.bin
   ```

## 测试与验证标准

我们使用 `logisim-bin` 提供的 C 编译产物（`mem.bin` 和 `sum.bin`）对处理器进行整机测试。验证模拟器逻辑是否完全正确的标准如下：

1. **执行周期结束**：程序 `mem.bin` 和 `sum.bin` 的预期运行时间均为 **6000 个周期**。模拟器内部需设定一个计数器，达到 6000 周期时停止执行。
2. **PC 指针检查**：运行结束时，PC 应停留在死循环代码段中。对照 `.txt` 反汇编文件，PC 必须位于 `halt` 函数附近。
3. **返回值检查**：程序结束时，寄存器 `a0` 的值必须严格为 `0`。
